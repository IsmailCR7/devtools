# devtools



## Правило веток

В этом проекте используется следующее именование веток:
*   **`master`** / **`main`** — основная стабильная ветка. Готовый код попадает сюда только после ревью и тестирования.
*   **`feature/DVT-X`** — ветка для разработки новой функциональности. `X` — это номер задачи в трекере (например, Jira-задача DVT-3).
    *   **Пример:** `feature/DVT-3` — ветка для разработки задачи под номером DVT-3.

**Порядок работы:**
1.  Создать новую ветку от `master`: `git checkout -b feature/DVT-3`
2.  Выполнить задачу, сделать коммиты.
3.  Отправить ветку на сервер и создать Pull Request (Merge Request) для слияния в `master`.
