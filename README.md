# DevTools

## Правила работы с ветками

В проекте используется следующая система веток:

### Основные ветки
* **`master`** — основная стабильная ветка. Готовый код попадает сюда только после проверки и тестирования через Pull Request.
* **`main`** — альтернативное название основной ветки (используется в зависимости от настроек репозитория).

### Ветки разработки
* **`feature/DVT-X`** — ветка для разработки новой функциональности. `X` — номер задачи в трекере (например, Jira-задача DVT-3).

### Примеры именования веток
* `master` — главная ветка
* `feature/DVT-3` — ветка для разработки задачи DVT-3
* `feature/DVT-4` — ветка для разработки задачи DVT-4

### Порядок работы
1. **Создание ветки** из `master`:
   ```bash
   git checkout master
   git pull origin master
   git checkout -b feature/DVT-X
